**Blogr Landing Page**

A solution to the [Blogr landing page challenge on Frontend Mentor](https://www.frontendmentor.io/challenges/blogr-landing-page-EX2RLAApP).

**Overview**

**The Challenge**

Users should be able to:

- View the optimal layout for the site depending on the device's screen size
- See hover states for all interactive elements on the page
- Toggle the mobile navigation menu open and closed
- Open and close dropdown menus in the navigation

**What I Learned**

The most challenging part of this project was getting the responsive layout right. The design has a lot of sections that shift between mobile and desktop, especially the two-column feature grids and the infrastructure section, which has rounded corners on opposite sides depending on the viewport. Getting those to move cleanly without breaking the visual integrity took a lot of iteration.

I also got more comfortable working with CSS `border-radius` to create the curved section shapes, and using `IntersectionObserver` in JavaScript to trigger scroll animations without a library.

One thing I would do differently next time is plan my breakpoints earlier in the process rather than adjusting them as issues came up.

**What I Would Improve**

Given more time, I would add smoother dropdown animations on desktop and improve keyboard accessibility across the navigation.
